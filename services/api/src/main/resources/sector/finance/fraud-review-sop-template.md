# Fraud / KYC flag — human review SOP (sample)

> **Sample SOP only.** Sector pack: `finance`.

## When human review is required

- Account freeze / restriction recommendations
- High-confidence fraud flags above policy threshold
- KYC escalations that would deny service access

## Reviewer checklist

1. Confirm model version and input freshness
2. Inspect false-positive history for similar profiles
3. Record decision and rationale in case system
4. Escalate true positives per AML / fraud playbooks

## Related controls

- `FIN_FRAUD_FALSE_POSITIVE_REVIEW`
- `FIN_CREDIT_EXPLAINABILITY`
